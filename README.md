# Финансовый Telegram-бот

Личный бот для учёта финансов. Google-таблица = база данных, бот = пульт к ней.
Вся математика в коде, нейросеть только формулирует советы.

## Что умеет
- 💰 Доход (такси/зарплата/родители/прочее) + автораспределение по долгам
- 💸 Расход по категориям, погашение долгов с уменьшением остатка
- 📊 Баланс и сколько свободно на жизнь
- 🍔 Остаток трат по категории за месяц
- ⚠️ Что горит (ближайшие платежи и просрочки)
- 🤖 Совет от нейросети на готовых цифрах
- 📋 Долги по приоритету погашения
- ⚙️ Настройки (оклад, аренда, связь — меняются на лету)
- Авто-начисление зарплаты 1 и 16 числа в 12:00
- Ежедневные напоминания о платежах (10:00)
- Ежедневный бэкап операций (03:00)

## Запуск (кратко)

1. Установить зависимости:
   ```
   pip install -r requirements.txt --break-system-packages
   ```

2. Создать `.env` из `.env.example` и заполнить:
   - `BOT_TOKEN` — от @BotFather
   - `OWNER_TELEGRAM_ID` — свой ID от @userinfobot
   - `LLM_API_KEY` — ключ ProxyAPI (опционально; без него «Совет» покажет сухие цифры)

3. Положить `credentials.json` (сервисный аккаунт Google) в папку проекта.
   Робот (email из credentials) должен быть добавлен в таблицу как **Редактор**.

4. Первый запуск (создаст листы в таблице):
   ```
   python3 bot.py
   ```

5. Автозапуск 24/7 через systemd:
   ```
   cp finance-bot.service /etc/systemd/system/
   systemctl daemon-reload
   systemctl enable finance-bot
   systemctl start finance-bot
   ```

## Полезные команды
- Логи: `journalctl -u finance-bot -f`
- Перезапуск: `systemctl restart finance-bot`
- Статус: `systemctl status finance-bot`

## Структура
- `bot.py` — точка входа, планировщик
- `config.py` — конфигурация из .env
- `sheets.py` — работа с Google Sheets
- `salary.py` — расчёт зарплаты, календарь 2026
- `debts.py` — приоритеты долгов
- `finance.py` — распределение дохода, балансы
- `llm.py` — советник через ProxyAPI
- `handlers.py` — обработчики кнопок
- `keyboards.py` — клавиатуры

## Надёжность
- Источник правды — Google-таблица (бот ничего не хранит локально)
- Каждая операция пишется сразу с подтверждением
- При ошибке Google — операция не теряется молча, бот сообщает
- retry (3 попытки) на все обращения к Google
- Идемпотентность авто-начислений (не задвоит зарплату)
- Ежедневный бэкап листа операций
